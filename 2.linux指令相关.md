# 符号链接（软链接）

是Linux下的**快捷方式**，符号链接将一个文件的路径以文件的形式拷贝到了另一个目录中

sudo ln -s [文件**绝对**路径] [目标路径]

sudo ln -s ~/download/Typora /usr/local/bin/typora

若是遇到上述情况，还需要在~/.bashrc 或者~/.profile目录中加上

`export PATH=$PATH:/usr/local/bin` —— 将该路径设置为PATH环境变量

`source ~/.bashrc`  启用环境变量

`sudo ln -s ~/zotero.desktop ~/.local/share/applications/zotero.desktop` -这个指令也就是说，如果一个软件有.desktop，那么可以软连接到桌面图片

硬链接就是ln src dest

![image-20250707142755543](https://raw.githubusercontent.com/upsetgrass/typora_pic_bed/main/image-20250707142755543.png)

类似于我们现在有一个文件A，其所存储的数据为D，软连接就是存储的文件A的路径，即用文件B指向文件A，实际还是利用文件A的各种环境信息，而硬链接就是用文件B指向数据D，那么此时文件B自己有一套环境，但是实际数据还是同一块，所以文件A的修改也会反映在B上，具体底层逻辑可以查看 [13.文件系统]

# strace

LInux工具如何运行？

1、看源码（source） 得知每一处静态细节、较为繁琐

2、看踪迹（trace） 了解运行动态行为，不全面，容易理解

踪迹工具strace（system call trace）记录程序运行过程中的系统调用信息：通过观察程序运行过程中发生的系统调用，可以大致了解程序关键行为

strace ls  查看指令ls的系统调用



ls如何被找到？

strace -f bash -c "ls"

-f：表示在 bash 进程中创建的任何子进程也将被跟踪。例如，如果 bash 启动了 ls，strace 会跟踪 ls 进程的系统调用。

bash：执行 bash 这个命令，这里是启动一个新的 Bash shell。

-c：让 bash 执行一个命令，并在执行完成后退出。它后面的字符串 "ls" 就是需要执行的命令。

"ls"：在 bash 中执行的命令

strace -f bash -c "abcde"  不管是什么指令，都会去寻找

| `gcc -E file.c -o file.i` | 只进行预处理，生成 `.i` 文件 |
| `gcc -S file.c -o file.s` | 生成汇编代码（`.s` 文件）    |
| `gcc -c file.c -o file.o` | 生成目标文件（.o），不链接   |
| `gcc file.o -o myprog`    | 将目标文件链接生成可执行文件 |

上表的-o不是必要的



优化

| 命令                | 说明                               |
| ------------------- | ---------------------------------- |
| `-O0`               | 无优化（默认）                     |
| `-O1`, `-O2`, `-O3` | 启用不同级别的优化（越大优化越强） |
| `-Os`               | 为减小代码体积而优化               |
| `-Ofast`            | 激进优化（可能不符合标准）         |

# dpkg

`dpkg` 的全称是 **Debian Package**，它是 **Debian 软件包管理系统的底层工具**

`dpkg` 负责**安装、删除、管理和查询**单个 `.deb` 格式的软件包文件



- 当你需要安装或查询一个**本地**下载的 `.deb` 文件时，使用 `dpkg`。
- 当你需要**安装软件及所有依赖**、**升级系统**或**搜索仓库**时，使用 `apt`。



| **常用命令** | **描述**                                              | **示例**                            |
| ------------ | ----------------------------------------------------- | ----------------------------------- |
| **安装**     | 安装一个本地的 `.deb` 软件包文件。                    | `sudo dpkg -i package.deb`          |
| **删除**     | 删除已安装的软件包。                                  | `sudo dpkg -r package-name`         |
| **彻底清除** | 删除软件包及其所有配置文件。                          | `sudo dpkg -P package-name`         |
| **列表**     | 列出系统中所有已安装的软件包。                        | `dpkg -l`                           |
| **查询文件** | 查找某个文件属于哪个已安装的软件包。                  | `dpkg -S /usr/bin/ld`               |
| **查询信息** | 查看一个已安装软件包的详细信息。                      | `dpkg -s package-name`              |
| **添加架构** | (如您之前所用) 添加对其他 CPU 架构的支持（如 i386）。 | `sudo dpkg --add-architecture i386` |





# libguestfs

`libguestfs` 是一个用于访问和修改虚拟机磁盘镜像（disk images）内部文件系统的 **C 库 (C library)** 和一套 **命令行工具集** 的总称。

**最小化虚拟机 (Appliance):** 运行一个 `libguestfs` 工具（比如 `virt-customize` 或 `virt-viewer`）时，它会在后台启动一个微小、临时的 QEMU 实例，称为 **Appliance**。



virt-customize：它的主要用途是**离线**（即虚拟机没有运行的情况下）对磁盘镜像文件（例如你使用的 `jammy-server-cloudimg-arm64.img`）进行各种自定义和修改。

| **类别**       | **常见操作示例**                                             |
| -------------- | ------------------------------------------------------------ |
| **系统配置**   | 设置 `root` 密码：`--root-password password:XXX`             |
| **用户管理**   | 添加/删除用户，设置用户密码和 SSH 密钥：`--ssh-inject`       |
| **文件操作**   | 上传文件到镜像内：`--upload local_file : /path/in/vm`        |

| **系统初始化** | 启用/禁用服务：`--enable-service sshd`                       |
| **执行命令**   | 在镜像内部的文件系统中执行 Shell 命令：`--run-command 'echo "Hello" >> /tmp/test'` |

# ar 解析deb

ar x XXX.deb

此时可以将该deb文件的debian-binary、control.tar.xz、data.tar.xz这三个文件解开

**`debian-binary`**: 包含版本号信息。

**`control.tar.xz` (或 `.gz`)**: 包含软件包的元数据（如依赖关系、安装脚本等）。

**`data.tar.xz` (或 `.zst`, `.gz`)**: **最重要的部分**，包含了软件包实际要安装的文件（可执行文件、配置文件、库文件等）。

我们很多需要的文件就都在data.tar.xz中
