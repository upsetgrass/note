# rust概要

Rust 速度惊人且内存利用率极高。由于没有运行时和垃圾回收，它能够胜任对性能要求特别高的服务，可以在嵌入式设备上运行，还能轻松和其他语言集成。



在 Rust 开发环境中，所有工具都安装在 `~/.cargo/bin` 目录中，您可以在这里找到包括 `rustc`、`cargo` 和 `rustup` 在内的 Rust 工具链。

## cargo

Cargo文档：https://doc.rust-lang.org/cargo/

cargo作为rust构建工具和包管理器的最新稳定版

cargo可以：

```shell
cargo build   # 可以构建项目
cargo run     # 可以运行项目
cargo test    # 可以测试项目
cargo doc     # 可以为项目构建文档
cargo publish # 可以将库发布到 crates.io
cargo init    # 创建Cargo.toml
```

`cargo build`会在target/debug/目录下创建对应名称的可执行文件，并会出现一个Cargo.lock，记录了本地所用依赖库的实际版本，这个文件又Cargo管理，不需要手动编辑

`cargo run` 做了两件事：编译+运行，如果发现src文件没有被修改，那么就是执行运行，不会重新编译，

`cargo check` 该指令能够快速检查代码保证其可以编译，但并不产生可执行文件，cargo check比cargo build会快很多

`cargo build --release` 发布构建时优化编译项目，会在target/release下生成可执行文件，这种方式的构建时间会更长，rust代码运行的更快。

## 基本概念

### 变量和可变性

#### 变量/常量

声明变量用let，声明常量用const

变量中有可变量和不可变量，可变量利用mut声明

const常量需要明确标注值的类型，不能对常量使用mut，并且常量只能设置为常量表达式，而不能是其他任何运行时计算出的值（关于常量：[常量求值](https://doc.rust-lang.org/reference/const_eval.html)），并且常量在声明它的作用域之中在整个程序生命周期中都有效。

```rust
let x = 1;  // 不可变量
let mut x = 2; // 可变量
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3; // 常量，将常量值绑定到一个名称上
```

#### 遮蔽（shadowing）

rust运行定义一个与之前变量同名的新变量，用新变量去遮蔽旧变量的值，遮蔽后使用该变量的行为会被视为使用第二个变量，直到第二个变量自己也被遮蔽或者第二个变量作用于结束。

```rust
fn main() {
    let x = 5;

    let x = x + 1;
    {
        let x = x * 2;
        println!("The value of x in the inner scope is: {x}"); // 12
    }
    println!("The value of x is: {x}"); // 6
}
```

遮蔽和将变量标记为mut有区别：

- 当不小心尝试对变量重新赋值时，如果没有使用 `let` 关键字，就会导致编译时错误。通过使用 `let`，我们可以用这个值进行一些计算，不过计算完之后变量仍然是不可变的。

- `mut` 与遮蔽的另一个区别是，当再次使用 `let` 时，实际上创建了一个新变量，我们可以改变值的类型，并且复用这个名字。这样能够使得程序更为简练（如果不使用遮蔽特性是不能改变变量的类型的）

  ```rust
  let spaces = "   ";
  let spaces = spaces.len(); // 无需定义spaces_str和spaces_len这两个变量
  ```

##### 元组

元组声明后长度固定

```
let tup: (i32, f64, u8) = (500, 5.2, 1);
```

元组中的元素顺序固定，可以利用.来访问

```rust
let tup: (i32, f64, u8) = (500, 5.2, 1);

let second_float = tup.1;

```

也可以将元组利用模式匹配（pattern matching）来解构（destructure）元组值

```rust

println!("The value of y is: {y}"); // 5.2
```

不带任何值的元组有个特殊的名称，叫做单元（unit）元组。这种值以及对应的类型都写作 `()`，表示空值或空的返回类型。如果表达式不返回任何其他值，则会隐式返回单元值。

对单元元祖的理解：

**rust中一切皆表达式**，在 C 或 Java 中，如果一个函数不返回任何东西，我们会用 `void`。但在 Rust 中，没有真正的“空”，只有“不携带信息的单元”

Rust 的函数和块级表达式{ ... } 必须返回点什么。如果你没有显式写 `return` 或者没用表达式结尾，Rust 就会默默地加一个 `()`

```rust
let x = { 1 + 2; }; // 1+2把表达式变成了语句，语句不返回值，所以返回一个()
println!("x:{:?}", x); // x:()
```



```rust
// 函数 A：最后一行是表达式，返回 i32
fn add_one(x: i32) -> i32 {
    x + 1
}

// 函数 B：最后一行是语句（带分号），返回 ()
fn add_one_silent(x: i32) {
    x + 1; 
}
```

### 控制流

根据条件是否为真来决定是否执行某些代码，以及根据条件是否为真来重复运行一段代码的能力是大部分编程语言的基本组成部分。Rust 代码中最常见的用来控制执行流的结构是 `if` 表达式和循环。

#### if

if表达式中的条件部分必须是布尔值，如果不是直接报错

```rust
let number = 3;

if number != 0 {
    println!("number was something other than zero");
}
```

需要注意的是if else if这种形式是多路选择一路，从上到下优先级递减，即找到一个符合的之后不会检查剩下的条件。

```rust
let number = 6;

if number % 4 == 0 {
    println!("number is divisible by 4");
} else if number % 3 == 0 {
    println!("number is divisible by 3");
} else if number % 2 == 0 {
    println!("number is divisible by 2");
} else {
    println!("number is not divisible by 4, 3, or 2");
}

```

注意，if结构是表达式，所以可以在let后面用if：

condition为true，则执行后面的{5} 5是数字，本身也是表达式，所以 number = 5

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { 6 };

    println!("The value of number is: {number}"); // 5
}
```

另外，对于if结构的不同分支的返回值都需要是相同的类型

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { "six" };
    println!("The value of number is: {number}");
}
```

还需要注意的是，如果if表达式的返回值不是()，那么是需要确定一个变量let x去接收的，不然会报错

```rust
fn main() { // err program
    if true{
        1
    }
    else{
        2
    }
    println!("hello");
}
```

if表达式的隐式则期待返回()

```rust
fn main() { // right program
    if true{
        1;
    }
    else{
        2;
    }
    println!("hello");
}
```

#### 循环执行

rust有：loop、while、for关键字来处理循环执行的情况

##### loop

loop关键字会不断的执行一段代码，只能通过break退出，常用于一个可能会失败需要重试的访问，网络请求...

```rust
fn main() {
    let mut counter = 0;

    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2; // 这里loop结构的返回值就是counter * 2
        }
    };

    println!("The result is {result}"); // 20
}
```

如果出现嵌套循环，需要使用循环标签以在多个循环之间消除歧义，在rust中嵌套循环的情况break和continue只能用于内层，对于一层循环上可以指定一个循环标签，然后将标签和break、continue一起使用

```rust
fn main() {
    let mut count = 0;// 外层的loop退出依靠count
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;// 内层的退出依靠

        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                break; // 内层退出：没执行两次退出一次内层
            }
            if count == 2 {
                break 'counting_up; // 外层退出
            }
            remaining -= 1;
        }

        count += 1;
    }
    println!("End count = {count}");
}
```

## 所有权

**所有权**（*ownership*）是 Rust 用于如何管理内存的一组规则。编译器在编译时会根据一系列的规则进行检查。如果违反了任何这些规则，程序都不能编译，并且在运行时，所有权系统的任何功能都不会减慢程序的运行。

### 栈和堆

栈和堆都是代码在运行中可供使用的内存。

栈空间的内存使用是先进后出，栈中的所有数据必须占用已知且固定的大小。在编译时大小未知或大小可能变化的数据需要存储到堆上。

堆的空间是缺乏组织的，内存分配器在堆的某处找到一块足够大的空位，将其标记为已使用，并返回一个表示该位置地址的指针，这个过程称为在堆上分配内存，简称“分配”。对于指向放入堆中数据的指针是已知的并且大小固定，可以将该指针存入栈中。

**入栈**比在堆上**分配**内存要快，这是因为入栈时无需为存储新数据去搜索内存空间，其入栈位置总是在栈顶。

**访问**堆上的数据比**访问**栈上的数据慢，因为必须通过指针来访问。

处理器在处理的数据彼此较近的时候（栈上数据）比 较远的时候（堆上数据）更高效。



当你的代码调用一个函数时，传递给函数的值（包括可能指向堆上数据的指针）和函数的局部变量被压入栈中。当函数结束时，这些值被移出栈。



跟踪哪部分代码正在使用堆上的哪些数据，最大限度的减少堆上的重复数据的数量，以及清理堆上不再使用的数据确保不会耗尽空间，这些问题正是所有权系统要处理的。

一旦理解了所有权，就不需要经常考虑栈和堆了，不过明白了所有权的主要目的就是管理堆数据，能够帮助解释为什么所有权要以这种方式工作。

### 所有权规则

- Rust 中的每一个值都有一个**所有者**（let x = 5;变量x就是5的所有者）

- 值在任一时刻有且只有一个所有者（处理重复释放的解决方案）

- 当所有者离开作用域，这个值将被丢弃（drop处理、栈指针移动）

### 内层与分配

对于String这些存储编译时未知大小的类型，需要把其数据存储到堆上，则

- 需要在运行时向内存分配器请求内存（分配）
- 需要一个处理完String后将内存返回给分配器的方法（回收）

这两条，前者是通过调用String::from来向内存分配器请求内存

后者，在rust中是在存储String这些类型的变量在离开其作用域时自动回收

```rust
{
	let s = String::from("hello"); // s此时有效
	// 使用s...
} // 离开s的所用于
// s不再有效
```

自然的，在当遇到}时，s就不再起作用，rust自动调用一个特殊的函数drop



这里会引出一个问题

#### 使用移动的变量与数据交互

```rust
let x = 5;
let y = x;
```

执行y = x之后，x还能正常使用

```rust
let s1 = String::from("hello");
let s2 = s1;
```

这种情况下，String类型指向的堆上开辟的实际内存空间并没有得到拷贝，s1和s2指向了同一块堆内存空间，而在rust中离开作用域会自动调用drop回收函数，那么必然会触发同一块堆空间释放两次的问题，所以在rust中只要遇到这种情况，在执行let s2 = s1;之后，s1就失效了，所以 rust 这里是一种 **移动** 而不是 **拷贝** 

在rust中是不会自动创建数据的深拷贝（深拷贝：深拷贝会对堆空间也拷贝一份，对效率影响较大  浅拷贝：只拷贝指针和管理数据比如len capacity这些栈上数据）



#### 不可达值的处理

```rust
let mut s = String::from("hello");
s = String::from("ahoy");

println!("{s}, world!"); // ahoy, world!
```

这种情况下执行s = String::from("ahoy");，hello的堆空间就没有指针指向它了，rust会自动调用drop回收hello的堆空间

#### 克隆变量

如果确实需要深拷贝，复制String中堆上的数据，而不仅仅是栈上数据，可以使用clone这个方法

```rust
let s1 = String::from("hello");
let s2 = s1.clone();
println!("s1 = {s1}, s2 = {s2}"); // s1 = hello, s2 = hello
```

#### trait

Trait是一种标签，一种特质，一个类型如果实现了某种Trait，代表这个类型的变量拥有某种能力，比如Copy Trait，rust给整形、浮点这些完全存储在栈上，不持有外部资源（堆内存、文件句柄）的类型打上了Copy trait，在进行赋值时，旧值不会被清除，而String就没有Copy Trait，所以旧值就会失效

```rust
let x = 5;
let y = x; // 有copy trait的存在，所以x在后续还能使用

println!("x = {x}, y = {y}");
```

在rust中，所有类型赋值默认都是Move形式，只有在给到Copy Trait才能让旧值还能正常使用

[如何给自定义类型添加Copy trait](https://kaisery.github.io/trpl-zh-cn/appendix-03-derivable-traits.html)

之前提到的自动调用drop，即时也是一种trait，String、vector这些变量都拥有drop trait，一个类型是不能同时拥有drop和copy trait的



常见的Trait

| **Trait 名称** | **赋予类型的“能力”**                                         |
| -------------- | ------------------------------------------------------------ |
| `Copy`         | 赋值时自动按位复制，旧变量不失效（如 `i32`, `bool`, `char`）。 |
| `Clone`        | 显式复制的能力。当你调用 `.clone()` 时，它会去复制数据。     |
| `Drop`         | 变量离开作用域时，自定义要执行的代码，常用于释放资源（堆内存、网络连接、文件句柄） |
| `Debug`        | 允许使用 `{:?}` 打印出来调试。                               |
| `PartialEq`    | 允许使用 `==` 进行比较。                                     |

### 所有权与函数

将值传递给函数也会涉及到移动或者是复制

```rust
fn main() {
    let s = String::from("hello");  // s 进入作用域

    takes_ownership(s);             // s 的值移动到函数里 ...
	println!("s = {s}");            // err，s在这里已经失效

    let x = 5;                      // x 进入作用域

    makes_copy(x);                  // x 应该移动函数里，
                                    // 但 i32 是 Copy 的，
    println!("{}", x);              // 所以在后面可继续使用 x

} // 这里，x 先移出了作用域，然后是 s。但因为 s 的值已被移走，
  // 没有特殊之处

fn takes_ownership(some_string: String) { // some_string 进入作用域
    println!("{some_string}");
} // 这里，some_string 移出作用域并调用 `drop` 方法。
  // 占用的内存被释放

fn makes_copy(some_integer: i32) { // some_integer 进入作用域
    println!("{some_integer}");
} // 这里，some_integer 移出作用域。没有特殊之处
```

### 返回值与作用域

```rust
fn main() {
    let s1 = gives_ownership();        // gives_ownership 将它的返回值传递给 s1

    let s2 = String::from("hello");    // s2 进入作用域

    some_string                        // 返回 some_string 并将其移至调用函数
}

// 该函数将传入字符串并返回该值
fn takes_and_gives_back(a_string: String) -> String {
    // a_string 进入作用域

    a_string  // 返回 a_string 并移出给调用的函数
}
```



```rust
fn main() {
    let s1 = String::from("hello");

    let (s2, len) = calculate_length(s1); // 返回元组，传回额外的数据

    println!("The length of '{s2}' is {len}.");
}

fn calculate_length(s: String) -> (String, usize) { // 如果这里不传回s，那么main中s1指向的堆空间就被回收了
    let length = s.len(); // len() 返回字符串的长度

    (s, length)
}
```

上面的例子发现，对于一个变量值，在不断的函数调用中，其所有权一直在转移，有时我们只想让函数使用某个值，而不获取所有权，那么就需要使用引用（references）

### 引用与借用

**引用**（*reference*）类似于指针，是一个地址，我们可以由此访问储存于该地址的属于其他变量的数据。与指针不同，引用在其生命周期内保证指向某个特定类型的有效值。

```rust
fn main() {
    let s1 = String::from("hello");

    let len = calculate_length(&s1); // 以对象的引用作为参数，而不需获得值的所有权

    println!("The length of '{s1}' is {len}."); // s1还能正常使用
}

fn calculate_length(s: &String) -> usize {
    s.len()
} // s是String的引用，s的所有权只控制了一个地址，并不拥有String实际值的所有权，所以不会对实际的s1造成影响
```

指向值s1的引用 - &s1

![image-20260225145040523](https://raw.githubusercontent.com/upsetgrass/typora_pic_bed/main/image-20260225145040523.png)

我们将创建一个引用的行为称为借用（borrowing），对于借用的值是无法被修改的

```rust
fn main() {
    let s = String::from("hello");

    change(&s);
}

fn change(some_string: &String) {
    some_string.push_str(", world"); // err
}
```

#### 可变引用

只需要利用mut关键字即可

```rust
fn main() {
    let mut s = String::from("hello"); // 可变变量

    change(&mut s); // 传入可变引用
}

fn change(some_string: &mut String) { // 函数签名接受可变引用
    some_string.push_str(", world"); // right
}
```

需要注意的是，对于一个变量的可变引用只能有一个

```rust
let mut s = String::from("hello");
let r1 = &mut s;
let r2 = &mut s; // err
println!("{r1}, {r2}");
```

这个机制是用于避免数据竞争：

- 多指针访问统一数据
- 至少由一个指针被用于写入
- 没有同步数据访问的机制

当然可以利用作用域来创建多个可变引用

```rust
fn main() {
    let mut s = String::from("hello");
    {
    	let r1 = &mut s;
      r1.push_str(", world");
    	println!("{r1}");	 // hello, world
    }
    {
    	let r2 = &mut s;// 这里的s是已经被r1修改过的
      r2.push_str(", linux"); 
    	println!("{r2}"); // hello, world, linux
    }
}
```

这种同一作用域中也不能同时存在可变和不可变引用

```rust
let mut s = String::from("hello");

    let r1 = &s; // right
    let r2 = &s; // right
    let r3 = &mut s; // err

    println!("{}, {}, and {}", r1, r2, r3);
```

对于 r1和r2在访问时并不希望有其他值能够改变他们访问的值，所以rust直接禁止了这种方式

#### 引用的作用域

一个引用的作用域从声明的地方开始一直持续到最后一次使用为止

```rust
let mut s = String::from("hello");

    let r1 = &s; // right
    let r2 = &s; // right
    println!("{r1} and {r2}");
    // 此位置之后 r1 和 r2 不再使用

    let r3 = &mut s; // right
    println!("{r3}");
```

#### 悬垂引用（Dangling References）

这是一种错误的引用形式，rust编译器能够在编译阶段就找出悬垂引用的错误

在具有指针的语言中，很容易通过释放内存时保留指向它的指针而错误地生成一个悬垂指针（dangling pointer）—— 指向可能已被分配给其他用途的内存位置的指针。

```rust
fn main() {
    let reference_to_nothing = dangle();
}

fn dangle() -> &String {
    let s = String::from("hello");

    &s // err，s离开dangle函数就被释放了，&s指向的内容被释放了
}


fn dangle_1() -> String {
	let s = String::from("Hello");
	s // right，所有权正常转移出去
}

```



### Slice类型

切片（Slice）用于**引用**一个集合内一段连续的元素序列，其使用一个中括号的[starting_index, ending_index]指定的range创建一个slice，starting_index是slice的第一个位置，ending_index是slice的最后一个位置的下一个位置

```rust
let s = String::from("hello world");

let hello = &s[0..5];
let world = &s[6..11];
```

对于hello和world就是slice引用类型，比前面的普通引用，多存了一个len

![image-20260225160809373](https://raw.githubusercontent.com/upsetgrass/typora_pic_bed/main/image-20260225160809373.png)

```
fn main() {
    let s = String::from("hello");  // s 进入作用域

    takes_ownership(s);             // s 的值移动到函数里 ...
    // println!("s={s}")            // err，s不再有效

    let x = 5;                      // x 进入作用域

    makes_copy(x);                  // x 应该移动函数里，
                                    // 但 i32 是 Copy 的，
    println!("{}", x);              // 所以在后面可继续使用 x

} // 这里，x 先移出了作用域，然后是 s。但因为 s 的值已被移走，
  // 没有特殊之处

fn takes_ownership(some_string: String) { // some_string 进入作用域
    println!("{some_string}");
} // 这里，some_string 移出作用域并调用 `drop` 方法。
  // 占用的内存被释放

fn makes_copy(some_integer: i32) { // some_integer 进入作用域
    println!("{some_integer}");
} // 这里，some_integer 移出作用域。没有特殊之处
```





额外的，对于range语法，如果starting_index从0开始，可以省略这个0，下面slice1、slice2是相同的含义，如果ending_index是len即最后一个字节，也可以舍弃尾部的数字，例如slice3和slice4

```rust
let s = String::from("hello");

let slice1 = &s[0..2];
let slice2 = &s[..2];

let slice3 = &s[3..len];
let slice4 = &s[3..];
```



slice对于一些操作主串种的子串的时候可以避免一些无效数据导致的错误

```rust
fn first_word1(s: &String) -> usize {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return i;
        }
    }
    s.len()
}


fn main() {
    let mut s = String::from("hello world");

    let word = first_word1(&s); // word 的值为 5

    s.clear(); // 这清空了字符串，使其等于 ""

    // word 在此处的值仍然是 5，
    // 但是没有更多的字符串让我们可以有效地应用数值 5。word 的值现在完全无效！
}
```

对于first_word1只能捕获一个String字符串中第一个单词的结尾位置，word接受这个整数值，此时如果String字符串被改变了，那么word就可能是一个错误的值，容易导致错误

而在first_word2中直接通过slice机制提取出子串，这个切片类型的引用就可以避免这种问题，借用原则明确在同一段作用于内不能同时存在可变和不可变引用，word是不可变引用，s.clear()中需要对s进行改变，需要是可变的，所以这里直接编译器报错

```rust
fn first_word2(s: &String) -> &str { // 这里的&str类型在栈上存了内存地址和一个长度，专用于Slice切片类型，&String是指向String对象的指针
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }

    &s[..]
}

fn main() {
    let mut s = String::from("hello world");

    let word = first_word2(&s);

    s.clear(); // err

    println!("the first word is: {word}");
}
```

fn first_word(s: &str) -> &str {
fn main() {
