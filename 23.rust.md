# rust概要

Rust 速度惊人且内存利用率极高。由于没有运行时和垃圾回收，它能够胜任对性能要求特别高的服务，可以在嵌入式设备上运行，还能轻松和其他语言集成。



在 Rust 开发环境中，所有工具都安装在 `~/.cargo/bin` 目录中，您可以在这里找到包括 `rustc`、`cargo` 和 `rustup` 在内的 Rust 工具链。

## cargo

Cargo文档：https://doc.rust-lang.org/cargo/

cargo作为rust构建工具和包管理器的最新稳定版

cargo可以：

```shell
cargo build   # 可以构建项目
cargo run     # 可以运行项目
cargo test    # 可以测试项目
cargo doc     # 可以为项目构建文档
cargo publish # 可以将库发布到 crates.io
cargo init    # 创建Cargo.toml
```

`cargo build`会在target/debug/目录下创建对应名称的可执行文件，并会出现一个Cargo.lock，记录了本地所用依赖库的实际版本，这个文件又Cargo管理，不需要手动编辑

`cargo run` 做了两件事：编译+运行，如果发现src文件没有被修改，那么就是执行运行，不会重新编译，

`cargo check` 该指令能够快速检查代码保证其可以编译，但并不产生可执行文件，cargo check比cargo build会快很多

`cargo build --release` 发布构建时优化编译项目，会在target/release下生成可执行文件，这种方式的构建时间会更长，rust代码运行的更快。

## 变量和可变性

### 变量/常量

声明变量用let，声明常量用const

变量中有可变量和不可变量，可变量利用mut声明

const常量需要明确标注值的类型，不能对常量使用mut，并且常量只能设置为常量表达式，而不能是其他任何运行时计算出的值（关于常量：[常量求值](https://doc.rust-lang.org/reference/const_eval.html)），并且常量在声明它的作用域之中在整个程序生命周期中都有效。

```rust
let x = 1;  // 不可变量
let mut x = 2; // 可变量
const THREE_HOURS_IN_SECONDS: u32 = 60 * 60 * 3; // 常量，将常量值绑定到一个名称上
```

### 遮蔽（shadowing）

rust运行定义一个与之前变量同名的新变量，用新变量去遮蔽旧变量的值，遮蔽后使用该变量的行为会被视为使用第二个变量，直到第二个变量自己也被遮蔽或者第二个变量作用于结束。

```rust
fn main() {
    let x = 5;

    let x = x + 1;
    {
        let x = x * 2;
        println!("The value of x in the inner scope is: {x}"); // 12
    }
    println!("The value of x is: {x}"); // 6
}
```

遮蔽和将变量标记为mut有区别：

- 当不小心尝试对变量重新赋值时，如果没有使用 `let` 关键字，就会导致编译时错误。通过使用 `let`，我们可以用这个值进行一些计算，不过计算完之后变量仍然是不可变的。

- `mut` 与遮蔽的另一个区别是，当再次使用 `let` 时，实际上创建了一个新变量，我们可以改变值的类型，并且复用这个名字。这样能够使得程序更为简练（如果不使用遮蔽特性是不能改变变量的类型的）

  ```rust
  let spaces = "   ";
  let spaces = spaces.len(); // 无需定义spaces_str和spaces_len这两个变量
  ```

## 数据类型

rust是静态类型的语言，即需要在编译时就知道所有变量的类型，rust会根据值及其使用方式推断想要使用的类型，当多种类型均有可能（“42”.parse()，将str→num），那么需要增加类型注解（`:u32`）

rust的数据类型分为标量类型（scalar）和复合类型（compound）

### 标量类型

rust有四种基本的标量类型：整形、浮点、布尔、字符

#### 整形

| 长度     | 有符号  | 无符号  |
| -------- | ------- | ------- |
| 8-bit    | `i8`    | `u8`    |
| 16-bit   | `i16`   | `u16`   |
| 32-bit   | `i32`   | `u32`   |
| 64-bit   | `i64`   | `u64`   |
| 128-bit  | `i128`  | `u128`  |
| 架构相关 | `isize` | `usize` |

有符号存储范围：$[-2^{n-1}, 2^{n-1}-1]$   无符号存储范围：$[0, 2^{n}-1]$

i8：[-128, 127]

u8：[0, 255]

isize和usize会根据运行程序的计算机架构来决定，对于64位架构上是64位，对于32位架构上是32位

| 数字字面值                    | 例子          |
| ----------------------------- | ------------- |
| Decimal (十进制)              | `98_222`      |
| Hex (十六进制)                | `0xff`        |
| Octal (八进制)                | `0o77`        |
| Binary (二进制)               | `0b1111_0000` |
| Byte (单字节字符)(仅限于`u8`) | `b'A'`        |

rust的默认类型通常是i32

对于整形溢出的问题：对于debug模式溢出会报错，但对于release模式溢出会截断（回绕）

可以显式处理整形溢出错误：wrapping\_* checked\_* ...

#### 浮点

浮点类型有f32和f64，默认是f64，现代CPU中，它与f32速度几乎一样，不过精度更高，所有浮点型都有符号

```rust
let x = 2.0;     // f64
let y: f32 = 2.0 // f32
```

#### 布尔

````rust
let t = true;
let f: bool = false;
````

#### 字符

用单引号声明char字面量，4字节，代表一个Unicode标量值，比ASCII表示更多的内容：带变音符号的字母（Accented letters），中文、日文、韩文等字符，emoji（绘文字）以及零长度的空白字符都是有效的 `char` 值，范围：U+0000-U+D7FF

用双引号声明字符串字面量，

```rust
	let c = 'z';
    let z: char = 'ℤ'; // with explicit type annotation
    let heart_eyed_cat = '😻';
```



#### 数值运算

加减乘除取余，整数出发会向零舍入到最接近的整数



### 复合类型

rust原生有：元祖和数组

#### 元组

元组声明后长度固定

```
let tup: (i32, f64, u8) = (500, 5.2, 1);
```

元组中的元素顺序固定，可以利用.来访问

```rust
let tup: (i32, f64, u8) = (500, 5.2, 1);
let first_num = tup.0;
let second_float = tup.1;
let third_num = tup.2;
```

也可以将元组利用模式匹配（pattern matching）来解构（destructure）元组值

```rust
let tup: (i32, f64, u8) = (500, 5.2, 1);
let (x, y, z) = tup;

println!("The value of y is: {y}"); // 5.2
```

不带任何值的元组有个特殊的名称，叫做单元（unit）元组。这种值以及对应的类型都写作 `()`，表示空值或空的返回类型。如果表达式不返回任何其他值，则会隐式返回单元值。

对单元元祖的理解：

**rust中一切皆表达式**，在 C 或 Java 中，如果一个函数不返回任何东西，我们会用 `void`。但在 Rust 中，没有真正的“空”，只有“不携带信息的单元”

Rust 的函数和块级表达式{ ... } 必须返回点什么。如果你没有显式写 `return` 或者没用表达式结尾，Rust 就会默默地加一个 `()`

```rust
let x = { 1 + 2; }; // 1+2把表达式变成了语句，语句不返回值，所以返回一个()
println!("x:{:?}", x); // x:()
```



```rust
// 函数 A：最后一行是表达式，返回 i32
fn add_one(x: i32) -> i32 {
    x + 1
}

// 函数 B：最后一行是语句（带分号），返回 ()
fn add_one_silent(x: i32) {
    x + 1; 
}
```



#### 数组类型

rust中的数组长度依旧是固定的，元素类型必须相同

```rust
let a1 = [1, 2, 3 , 4];
let a2: [i32:5] = [1, 2, 3, 4, 5]; // [元素类型:元素个数]
let a3 = [3; 2]; // 与 a3 = [3, 3];效果相同
```

数组是可以在栈 (stack) 上分配的已知固定大小的单个内存块。可以使用索引来访问数组的元素

```rust
let a = [1, 2, 3, 4, 5];

let first = a[0]; // 1
let second = a[3]; // 4
```

在rust中，对数组元素进行a[i]进行访问时，会对i进行检查，如果超出索引返回，会报panic-程序会因错误退出——运行时错误 （rust安全原则）



## 函数

rust代码的函数命名采用snake case，即所有字母都是小写并使用下划线分隔单词

```rust
fn main() {
    another_function(5);
}
fn another_function(x: i32) {
    println!("The value of x is: {x}");
}
```

在rust中，函数必须明确每一个参数的类型

```rust
fn main() {
    print_labeled_measurement(5, 'h');
}

fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("The measurement is: {value}{unit_label}");
}
```

### 语句和表达式

`let y = 5; ` 是一个语句，语句不返回值，因此不能把语句赋值给另一个变量，即：

```rust
let x = (let y = 3); // err

let x = {let y = 3; println!("y={y}");}; // right,这里y在{}范围内需要被用到，不然报warning

```

函数调用是表达式、宏调用是表达式、大括号创建的新的块作用域是表达式

```rust
fn main() {
    let y = {
        let x = 3;
        x + 1 // 返回，注意没有; x+1是表达式， x+1;就会变成语句，则导致y为()
    };
    println!("The value of y is: {y}"); // 4
}
```

### 函数返回值

rust中函数返回值等同于函数体最后一个表达式的值，当然也可以使用return关键字进行返回。

在函数定义部分需要注意分号的使用，如果最后没有表达式而是语句并且函数的返回类型是i32而不是()

```rust
fn five() -> i32 {
    5
}

fn main() {
    let x = five();

    println!("The value of x is: {x}");
}
```



## 控制流

根据条件是否为真来决定是否执行某些代码，以及根据条件是否为真来重复运行一段代码的能力是大部分编程语言的基本组成部分。Rust 代码中最常见的用来控制执行流的结构是 `if` 表达式和循环。

### if

if表达式中的条件部分必须是布尔值，如果不是直接报错

```rust
let number = 3;

if number != 0 {
    println!("number was something other than zero");
}
```

需要注意的是if else if这种形式是多路选择一路，从上到下优先级递减，即找到一个符合的之后不会检查剩下的条件。

```rust
let number = 6;

if number % 4 == 0 {
    println!("number is divisible by 4");
} else if number % 3 == 0 {
    println!("number is divisible by 3");
} else if number % 2 == 0 {
    println!("number is divisible by 2");
} else {
    println!("number is not divisible by 4, 3, or 2");
}

```

注意，if结构是表达式，所以可以在let后面用if：

condition为true，则执行后面的{5} 5是数字，本身也是表达式，所以 number = 5

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { 6 };

    println!("The value of number is: {number}"); // 5
}
```

另外，对于if结构的不同分支的返回值都需要是相同的类型

```rust
fn main() {
    let condition = true;
    let number = if condition { 5 } else { "six" };
    println!("The value of number is: {number}");
}
```

还需要注意的是，如果if表达式的返回值不是()，那么是需要确定一个变量let x去接收的，不然会报错

```rust
fn main() { // err program
    if true{
        1
    }
    else{
        2
    }
    println!("hello");
}
```

if表达式的隐式则期待返回()

```rust
fn main() { // right program
    if true{
        1;
    }
    else{
        2;
    }
    println!("hello");
}
```

### 循环执行

rust有：loop、while、for关键字来处理循环执行的情况

#### loop

loop关键字会不断的执行一段代码，只能通过break退出，常用于一个可能会失败需要重试的访问，网络请求...

```rust
fn main() {
    let mut counter = 0;

    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2; // 这里loop结构的返回值就是counter * 2
        }
    };

    println!("The result is {result}"); // 20
}
```

如果出现嵌套循环，需要使用循环标签以在多个循环之间消除歧义，在rust中嵌套循环的情况break和continue只能用于内层，对于一层循环上可以指定一个循环标签，然后将标签和break、continue一起使用

```rust
fn main() {
    let mut count = 0;// 外层的loop退出依靠count
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;// 内层的退出依靠

        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                break; // 内层退出：没执行两次退出一次内层
            }
            if count == 2 {
                break 'counting_up; // 外层退出
            }
            remaining -= 1;
        }

        count += 1;
    }
    println!("End count = {count}");
}
```

#### while

```rust
fn main() {
    let mut number = 3;

    while number != 0 {
        println!("{number}!");

        number -= 1;
    }

    println!("LIFTOFF!!!");
}
```

#### for

```rust
fn main() {
    let a = [10, 20, 30, 40, 50];

    for element in a {
        println!("the value is: {element}"); 10 20 30 40 50
    }
}
```

下例使用标准库提供的Range类型，用来生成从一个数字到另一个数字**之前**结束的所有数字序列，又利用了rev()翻转了序列，完成了倒计时

```rust
fn main() {
    for number in (1..4).rev() {
        println!("{number}!"); // 3 2 1
    }
    println!("LIFTOFF!!!");
}
```



## 所有权

**所有权**（*ownership*）是 Rust 用于如何管理内存的一组规则。编译器在编译时会根据一系列的规则进行检查。如果违反了任何这些规则，程序都不能编译，并且在运行时，所有权系统的任何功能都不会减慢程序的运行。

### 栈和堆

栈和堆都是代码在运行中可供使用的内存。

栈空间的内存使用是先进后出，栈中的所有数据必须占用已知且固定的大小。在编译时大小未知或大小可能变化的数据需要存储到堆上。

堆的空间是缺乏组织的，内存分配器在堆的某处找到一块足够大的空位，将其标记为已使用，并返回一个表示该位置地址的指针，这个过程称为在堆上分配内存，简称“分配”。对于指向放入堆中数据的指针是已知的并且大小固定，可以将该指针存入栈中。

**入栈**比在堆上**分配**内存要快，这是因为入栈时无需为存储新数据去搜索内存空间，其入栈位置总是在栈顶。

**访问**堆上的数据比**访问**栈上的数据慢，因为必须通过指针来访问。

处理器在处理的数据彼此较近的时候（栈上数据）比 较远的时候（堆上数据）更高效。



当你的代码调用一个函数时，传递给函数的值（包括可能指向堆上数据的指针）和函数的局部变量被压入栈中。当函数结束时，这些值被移出栈。



跟踪哪部分代码正在使用堆上的哪些数据，最大限度的减少堆上的重复数据的数量，以及清理堆上不再使用的数据确保不会耗尽空间，这些问题正是所有权系统要处理的。

一旦理解了所有权，你就不需要经常考虑栈和堆了，不过明白了所有权的主要目的就是管理堆数据，能够帮助解释为什么所有权要以这种方式工作。

### 所有权规则

- Rust 中的每一个值都有一个**所有者**

- 值在任一时刻有且只有一个所有者

- 当所有者离开作用域，这个值将被丢弃
