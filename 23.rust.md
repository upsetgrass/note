# rust概要

Rust 速度惊人且内存利用率极高。由于没有运行时和垃圾回收，它能够胜任对性能要求特别高的服务，可以在嵌入式设备上运行，还能轻松和其他语言集成。



在 Rust 开发环境中，所有工具都安装在 `~/.cargo/bin` 目录中，您可以在这里找到包括 `rustc`、`cargo` 和 `rustup` 在内的 Rust 工具链。

## cargo

Cargo文档：https://doc.rust-lang.org/cargo/

cargo作为rust构建工具和包管理器的最新稳定版

cargo可以：

```shell
cargo build   # 可以构建项目
cargo run     # 可以运行项目
cargo test    # 可以测试项目
cargo doc     # 可以为项目构建文档
cargo publish # 可以将库发布到 crates.io
cargo init    # 创建Cargo.toml
```

`cargo build`会在target/debug/目录下创建对应名称的可执行文件，并会出现一个Cargo.lock，记录了本地所用依赖库的实际版本，这个文件又Cargo管理，不需要手动编辑

`cargo run` 做了两件事：编译+运行，如果发现src文件没有被修改，那么就是执行运行，不会重新编译，

`cargo check` 该指令能够快速检查代码保证其可以编译，但并不产生可执行文件，cargo check比cargo build会快很多

`cargo build --release` 发布构建时优化编译项目，会在target/release下生成可执行文件，这种方式的构建时间会更长，rust代码运行的更快。







## crates包依赖

在rust中将包称为crates，rust语言是社区驱动型语言，在crates.io作为rust中心化平台，存储了全球rust开发者的rust源代码以供下载使用，其也会存储开发者的历史代码版本，确定依赖项是统一稳定可靠的。

https://crates.io/

在rust项目中的Cargo.toml的[dependencies]下添加依赖：

```
[dependencies]
ferris-says = "0.3.1"
```

然后运行`cargo build  ` 后续即可使用 ferris-says 库，然后



在.rs文件中，添加

`use ferris_says::say;` 就可以使用ferris_says crate中导出的say函数了



`cargo doc --open` 命令用于构建所有本地依赖提供的文档并在浏览器中打开





# 安装/卸载

`curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh` // 利用rustup工具下载rust

rustup作为rust安装器和版本管理工具，可以利用`rustup update`更新到rust最新版本

卸载rust：

`rustup self uninstall`
